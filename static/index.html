GUI for check_httpv2

A small local web GUI to run and schedule check_httpv2 checks.

Project files
  • app.py — Flask backend (runs checks, scheduler, CSV/JSON logging)
  • static/index.html — UI (tabs, forms, modals)
  • static/script.js — UI logic (profiles, summary parsing, scheduler UI)
  • static/style.css — UI styles
  • bin/check-http (Linux/macOS) / bin/check-http.exe (Windows) — the check_httpv2 binary
Generated files
  • profiles.json — exported profiles (synced from browser LocalStorage)
  • results.csv — appended run history (for Grafana etc.)
  • last_<profile>.json — latest result per profile (used for live updates)
Tip: If you move the app directory, keep these files in the same folder as app.py.
Quick start
  1. Create a virtual environment and install dependencies (Ubuntu examples). 
    python3 -m venv .venv
source .venv/bin/activate

# Option A: (recommended) install minimal runtime deps (Flask)
pip install flask

# Option B: install from a pinned freeze file (will be added)
pip install -r requirements-freeze.txt

# Make check_httpv2 executable
chmod +x bin/check-http
  2. Start the web app: 
    python3 app.py
  3. Open the GUI in your browser (see the terminal output for the exact URL).
Notes:
  • The GUI stores profiles in the browser and syncs them to profiles.json on the server.
  • The scheduler runs one background thread per started profile.
  • results.csv is served at /results.csv for easy consumption.
